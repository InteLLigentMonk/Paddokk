You are helping the user create a semantic version release for the Paddokk project.

## Pre-flight Checks

1. **Verify on main branch:**
   ```bash
   git branch --show-current
   ```
   If not on `main`, stop and inform the user they must be on main to create a release.

2. **Verify clean working tree:**
   ```bash
   git status --porcelain
   ```
   If there are uncommitted changes, stop and inform the user to commit or stash changes first.

3. **Ensure latest from remote:**
   ```bash
   git fetch origin
   git status
   ```
   Check if local main is behind remote. If behind, inform user to pull first.

## Show Release Preview

1. **Show commits since last tag:**
   ```bash
   git log $(git describe --tags --abbrev=0 2>/dev/null || echo "HEAD")..HEAD --oneline --no-decorate
   ```
   If no tags exist, show all commits. Display these to the user.

2. **Run dry-run release:**
   ```bash
   cd client && npm run release:dry
   ```
   This shows what version will be bumped and preview of CHANGELOG. Display the output to the user.

## User Confirmation

Ask the user to confirm the release with AskUserQuestion:

**Question:**
- "Ready to create release?"
- Options:
  - "Yes, create release" → Proceed
  - "No, cancel" → Stop and exit
  - "Override version bump" → Ask which version (major/minor/patch)

If user wants to override:
- Ask: "Which version bump?"
  - "Major (breaking changes)"
  - "Minor (new features)"
  - "Patch (bug fixes)"

## Execute Release

1. **Run the release command:**

   If automatic bump:
   ```bash
   cd client && npm run release
   ```

   If user specified version:
   ```bash
   cd client && npm run release:major    # or minor, or patch
   ```

2. **Verify release was created:**
   ```bash
   git tag --points-at HEAD
   git log -1 --oneline
   ```
   Show the user the created tag and commit.

3. **Push to remote with tags:**
   ```bash
   git push --follow-tags
   ```

## Success Message

Show the user:
```
✅ Release created successfully!

Version: {version}
Tag: v{version}
Commits included: {count}

Changes pushed to remote. View the release on GitHub:
https://github.com/InteLLigentMonk/Paddokk/releases/tag/v{version}

CHANGELOG.md has been updated with release notes.
```

## Error Handling

If any step fails:
1. Show the error message
2. Explain what went wrong
3. Suggest how to fix it
4. Do NOT continue with subsequent steps

Common errors:
- **Not on main:** User must checkout main first
- **Uncommitted changes:** User must commit or stash
- **No commits since last tag:** Cannot create release with no changes
- **Push failed:** May need to pull first if remote has changes

## Important Notes

- Always verify branch and working tree status first
- Show dry-run output before making changes
- Require user confirmation before creating release
- Push with `--follow-tags` to include the version tag
- The release commit is auto-generated by `commit-and-tag-version`
